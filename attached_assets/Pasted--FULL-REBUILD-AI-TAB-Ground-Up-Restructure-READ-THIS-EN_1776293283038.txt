## MANDATORY MACRO PRE-FLIGHT (NEW — Critical Addition)

Before ANY AI call (Trade Ideas, Quant Scan, or Ask AI analysis), the system MUST fetch and display live macro events. This is a non-negotiable step.

### Backend: New Macro Pre-Flight Endpoint

Create a new route `GET /api/macro/preflight` that:

1. **Pulls from INTERNAL sources (already available):**
- `macroEvents` from DataBus (the existing macro calendar data)
- `killSwitch` status from DataBus
- News feed items already in the app (political alpha, conflict events, SEC insider buying)
1. **Pulls from EXTERNAL sources (new — use existing Finnhub news or add a free macro API):**
- Check Finnhub’s `/news` endpoint (you already have a Finnhub API key) for breaking market news in the last 2 hours
- Filter for HIGH IMPACT keywords: “FOMC”, “CPI”, “NFP”, “rate decision”, “rate cut”, “rate hike”, “tariff”, “sanctions”, “Fed”, “ECB”, “BOJ”, “BOE”, “BOC”, “GDP”, “inflation”, “employment”, “payrolls”, “Powell”, “Lagarde”, “trade war”, “default”, “emergency”
- If Finnhub news is not available, fall back to internal DataBus macroEvents only
1. **Returns a structured macro context object:**

```json
{
  "timestamp": "2026-04-15T16:30:00-04:00",
  "status": "CLEAR|CAUTION|BLOCKED",
  "killSwitch": false,
  "eventsNext2H": [],
  "eventsNext4H": [
    {"event": "BOC Rate Decision", "time": "09:45 ET", "impact": "HIGH", "status": "PASSED"}
  ],
  "eventsNext24H": [
    {"event": "FOMC Williams Speech", "time": "08:35 ET Apr 16", "impact": "MEDIUM"}
  ],
  "breakingNews": [
    {"headline": "Trump threatens to fire Fed Chair Powell", "source": "Finnhub", "time": "2H ago", "impact": "HIGH", "affectedAssets": ["DXY", "XAU", "BTC"]}
  ],
  "activeConflicts": ["Iran tensions — energy impact"],
  "summary": "Macro clear for next 2H. BOC decision passed. Williams speech tomorrow pre-market. Political risk elevated (Fed chair threat)."
}
```

1. **Cache this for 5 minutes** — don’t re-fetch on every AI call

### Frontend: Macro Pre-Flight Bar

Display the macro status as a visible bar ABOVE the trade ideas and scan results. Users should see what was checked:

```
┌──────────────────────────────────────────────────────────────┐
│  📡 MACRO PRE-FLIGHT — Checked 4:30 PM ET                   │
│  ✅ No high-impact events in next 2H                        │
│  ⚠️ FOMC Williams speech tomorrow 08:35 ET                  │
│  🔴 Breaking: Trump threatens to fire Fed Chair Powell      │
│     → Affects: USD, Gold, BTC                               │
└──────────────────────────────────────────────────────────────┘
```

**Status colors:**

- ✅ `CLEAR` (green): No high-impact events within 2H, no breaking news
- ⚠️ `CAUTION` (amber): Events within 4H or medium-impact breaking news. AI dampens confidence 20%.
- 🚫 `BLOCKED` (red): High-impact event within 2H or kill switch active. AI will NOT generate new signals — shows message: “Macro blocked — [event] in [time]. Signals paused until [time].”

### Integration Into AI Calls

**BEFORE calling any AI endpoint**, the frontend must:

1. Call `GET /api/macro/preflight`
1. Display the macro bar to the user
1. If status is `BLOCKED`, do NOT call the AI — show the block message instead
1. If status is `CLEAR` or `CAUTION`, inject the full macro context into the AI prompt as a new section:

```
MACRO PRE-FLIGHT (LIVE — checked {timestamp}):
Status: {CLEAR|CAUTION|BLOCKED}
Events next 2H: {list or "None"}
Events next 4H: {list or "None"}
Events next 24H: {list}
Breaking news: {list or "None"}
Active geopolitical risks: {list or "None"}

INSTRUCTIONS: Factor this macro context into EVERY signal. If CAUTION, reduce conviction by 20% and compress TP by 25%. Note the specific macro risk in the "flags" field of each trade. If breaking news affects a specific asset, either skip that asset or flag it prominently.
```

This section gets injected into the user message sent to Claude, right BEFORE the market data sections (Section A, B, C).

### Why This Matters

The current system has stale macro calendar data that refreshes every 30 seconds via DataBus, but it doesn’t catch:

- Breaking news (tariff announcements, geopolitical escalations, emergency Fed actions)
- Events that just happened in the last hour and are still being priced in
- News that affects specific assets the user is about to trade

The pre-flight check ensures the AI is NEVER generating signals blind to what just happened in the world.

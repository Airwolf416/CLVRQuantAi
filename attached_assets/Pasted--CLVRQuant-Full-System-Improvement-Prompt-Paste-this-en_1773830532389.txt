## FIX 1 — MACRO TAB: REMOVE DUPLICATE EVENTS

The macro calendar is showing duplicate entries when the same event appears in multiple sources (e.g. ForexFactory AND Central Banks both return the FOMC Rate Decision).

**Fix:** Before rendering the events list, deduplicate using a composite key:

```javascript
const deduplicatedEvents = Object.values(
  rawEvents.reduce((acc, event) => {
    const key = `${event.title.toLowerCase().trim()}_${event.date}_${event.time}`;
    if (!acc[key]) {
      acc[key] = event;
    } else {
      // Merge: keep the entry with more data (prefer non-null forecast/previous)
      if (!acc[key].forecast && event.forecast) acc[key].forecast = event.forecast;
      if (!acc[key].previous && event.previous) acc[key].previous = event.previous;
    }
    return acc;
  }, {})
);
```

After deduplication, only one FOMC entry should appear with the most complete data (forecast + previous populated).

**System prompt addition for the Brief generation call:**

Add this block to the brief generation system prompt:

```
CRITICAL RULE — PENDING EVENTS:
Before writing the brief, check the macro event list for any HIGH impact events scheduled within the next 24 hours that have status = PENDING (no actual result yet).

For any PENDING event:
- You MUST NOT state or imply the outcome
- You MUST NOT write phrases like "Fed holds", "CPI came in at", "rate decision confirmed", or any language that describes a result
- You MUST write forward-looking language such as:
  "FOMC Rate Decision is scheduled for 14:00 ET today — market is pricing in a hold at 4.25-4.50%. Watch for forward guidance and dot plot revisions as the key market mover."

Violation of this rule produces misleading information that could cause financial harm. It is your most important constraint when generating the brief.

When an event has status = RELEASED and actual result is populated, you may then describe the outcome using the actual value provided. Never infer — only state confirmed data.
```

## FIX 5 — DATA SYNC ACROSS ALL TABS

All tabs (Macro, Brief, Signals, Markets, Radar) must share a single data context object that is fetched once per refresh cycle and passed to every component. This prevents inconsistencies where one tab shows a different price than another.

**Implementation:**

```javascript
// Single shared data store — refresh every 60 seconds
class MarketDataStore {
  constructor() {
    this.data = null;
    this.lastFetched = null;
    this.subscribers = [];
  }

  async refresh() {
    const [prices, funding, macro, volume] = await Promise.all([
      fetchPrices(),       // Binance / Finnhub
      fetchFunding(),      // Binance perpetuals
      fetchMacroEvents(),  // ForexFactory + Central Banks (deduplicated)
      fetchVolume(),       // 24h volume + 7-day avg
    ]);

    this.data = {
      prices,
      funding,
      macro: deduplicateEvents(macro), // Apply Fix 1 here
      volume,
      fetched_at: new Date().toISOString(),
    };

    this.lastFetched = Date.now();
    this.notify();
  }

  subscribe(callback) {
    this.subscribers.push(callback);
  }

  notify() {
    this.subscribers.forEach(cb => cb(this.data));
  }
}

// All tabs subscribe to the same store
const store = new MarketDataStore();
store.refresh();
setInterval(() => store.refresh(), 60000);

// Each tab
store.subscribe((data) => {
  updateMacroTab(data);
  updateBriefTab(data);
  updateSignalsTab(data);
  updateMarketsTab(data);
});
```

This ensures the AI (Claude) always receives the exact same data context regardless of which tab triggered the analysis call.

### 6b — Pump detection threshold upgrade

Replace the current single-threshold trigger with a multi-factor confirmation system:

```javascript
function isPumpSignal(token) {
  const checks = {
    // 1. Price move must be significant relative to session volatility
    priceMove: token.change5min >= 1.5,              // Minimum 1.5% in 5min (was 0.8%)

    // 2. Volume must confirm the move (not just price drift)
    volumeSpike: token.volume5min >= token.avgVolume5min * 3.0,  // 3x average volume

    // 3. OI must be meaningful (filters micro-cap noise)
    minOI: token.openInterest >= 10_000_000,          // $10M minimum OI (was $3M)

    // 4. Funding must not be extremely elevated (prevents buying into squeeze top)
    fundingHealthy: token.fundingRate <= 0.003,       // Max 0.003%/8h

    // 5. Move must be sustained (not already fading)
    notFading: token.change1min >= 0,                 // Still moving up in last 1min

    // 6. For dumps: inverse of above checks
  };

  const passedChecks = Object.values(checks).filter(Boolean).length;
  const totalChecks = Object.keys(checks).length;
  const confidence = Math.round((passedChecks / totalChecks) * 100);

  return {
    isSignal: passedChecks >= 4,   // Must pass at least 4/5 checks
    confidence,
    failedChecks: Object.entries(checks)
      .filter(([, v]) => !v)
      .map(([k]) => k),
  };
}
```

Show which checks passed and which failed on each signal card:

## FIX 1 — FOMC COUNTDOWN: HIDE OR UPDATE AFTER EVENT HAS PASSED

**Bug:** The “Next Macro Event” countdown is still showing and counting down after the FOMC Rate Decision at 14:00 ET has already passed. The timer should never show a negative or post-event countdown.

**Fix — update the countdown logic with these rules:**

```javascript
function renderNextMacroEvent(events) {
  const now = new Date();

  // Filter to only FUTURE events (strictly after current time)
  const upcomingEvents = events.filter(event => {
    const eventTime = new Date(event.datetime); // must be in ISO format with timezone
    return eventTime > now;
  });

  const nextEventSection = document.getElementById('next-macro-event'); // adjust to your selector

  if (upcomingEvents.length === 0) {
    // No upcoming events — hide the section entirely
    nextEventSection.style.display = 'none';
    return;
  }

  // Sort ascending and take the next one
  upcomingEvents.sort((a, b) => new Date(a.datetime) - new Date(b.datetime));
  const nextEvent = upcomingEvents[0];

  // Render the countdown for the next genuine future event
  renderCountdown(nextEvent);
}

function renderCountdown(event) {
  const countdownInterval = setInterval(() => {
    const now = new Date();
    const eventTime = new Date(event.datetime);
    const diff = eventTime - now;

    if (diff <= 0) {
      // Event has now passed — stop countdown, update status to PENDING → check for result
      clearInterval(countdownInterval);
      updateEventStatus(event.id, 'RELEASED');  // triggers your result polling (Fix 2 from previous prompt)
      renderNextMacroEvent(getAllEvents());       // re-render to show next event or hide
      return;
    }

    const hours   = Math.floor(diff / 1000 / 60 / 60);
    const minutes = Math.floor((diff / 1000 / 60) % 60);
    const seconds = Math.floor((diff / 1000) % 60);

    document.getElementById('countdown-display').textContent =
      `${String(hours).padStart(2, '0')} : ${String(minutes).padStart(2, '0')} : ${String(seconds).padStart(2, '0')}`;
  }, 1000);
}
```

**Key rule:** Event datetime must be stored in UTC/ISO format with timezone offset included (e.g. `2026-03-18T14:00:00-05:00` for 14:00 ET). Never store as a naive time string — this causes timezone bugs where events appear to not have passed yet.

**After the event passes:**

- The countdown section either disappears or immediately advances to the next upcoming event
- The event card in the Macro tab updates from `PENDING` to `RELEASED` and begins polling for the actual result (see previous prompt Fix 2)
- The Alerts tab fires a notification: “FOMC Rate Decision released — checking for result…”

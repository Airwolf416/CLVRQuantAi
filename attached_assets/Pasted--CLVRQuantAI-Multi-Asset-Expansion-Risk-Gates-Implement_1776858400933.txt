## 2. MACRO KILL SWITCH (halt longs on BTC flush)

**File: `server/lib/circuitBreaker.ts`**

Add a new global gate:

```typescript
// Returns true if we should halt ALL long signals globally
export async function isMacroRiskOff(): Promise<{
  halted: boolean;
  reason?: string;
}> {
  // Get BTC price 4 hours ago vs now
  const btcCandles = await getBTC4hCandles(); // use existing HL/Binance fetcher
  if (!btcCandles || btcCandles.length < 2) return { halted: false };
  
  const priceNow = btcCandles[btcCandles.length - 1].close;
  const price4hAgo = btcCandles[0].close;
  const pctChange = ((priceNow - price4hAgo) / price4hAgo) * 100;
  
  if (pctChange <= -3.0) {
    return {
      halted: true,
      reason: `Macro risk-off: BTC ${pctChange.toFixed(2)}% in 4h`,
    };
  }
  return { halted: false };
}
```

**File: `server/routes.ts`**

In `/api/quant` and the background signal loop, call this BEFORE generating any LONG signal. If `halted=true`, skip LONG generation entirely and log the rejection to `rejectionLog.ts` with the reason. SHORTs still allowed.

## 3. PER-ASSET COOLDOWN (2h min between signals on same token+direction)

**File: `server/lib/adaptiveThresholds.ts`** (or new file `server/lib/cooldown.ts`)

```typescript
// Query ai_signal_log for last signal on this (token, direction)
export async function isInCooldown(
  token: string,
  direction: "LONG" | "SHORT"
): Promise<{ inCooldown: boolean; minutesLeft?: number }> {
  const COOLDOWN_MINUTES = 120;
  const lastSignal = await db
    .select()
    .from(aiSignalLog)
    .where(and(eq(aiSignalLog.token, token), eq(aiSignalLog.direction, direction)))
    .orderBy(desc(aiSignalLog.createdAt))
    .limit(1);
  
  if (!lastSignal.length) return { inCooldown: false };
  
  const minutesSince = (Date.now() - new Date(lastSignal[0].createdAt).getTime()) / 60000;
  if (minutesSince < COOLDOWN_MINUTES) {
    return { inCooldown: true, minutesLeft: Math.ceil(COOLDOWN_MINUTES - minutesSince) };
  }
  return { inCooldown: false };
}
```

**File: `server/routes.ts`**

Call `isInCooldown()` at the start of signal generation per (token, direction). If in cooldown, skip that combo and log rejection with "cooldown active, {X} min left".

This fixes the ONDO-spam issue (4 signals in 3 hours).

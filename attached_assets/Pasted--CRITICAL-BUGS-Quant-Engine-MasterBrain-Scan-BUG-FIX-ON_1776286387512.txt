## BUG 5: Mobile BTC scan still returns “AI returned malformed data”

On mobile, scanning BTC returns “BTC: AI returned malformed data” with 0 signals found.

### Fix

1. Check if the mobile API call uses the same endpoint and prompt as desktop. If there’s a separate mobile handler, it may not have been updated with the new prompt.
1. The most likely cause: the AI is returning markdown or prose instead of JSON, and the parser is failing. Ensure the system prompt includes:

```
You MUST respond with valid JSON only. No markdown. No prose. No preamble. No code fences. Start your response with { or [ and end with } or ]. Any non-JSON text will cause a parsing error.
```

1. Add a fallback in the response parser — if JSON parsing fails, try to extract JSON from the response by finding the first `{` and last `}`:

```javascript
function extractJSON(text) {
  // Try direct parse first
  try { return JSON.parse(text); } catch(e) {}
  
  // Strip markdown fences
  const stripped = text.replace(/```json\n?/g, '').replace(/```\n?/g, '').trim();
  try { return JSON.parse(stripped); } catch(e) {}
  
  // Find JSON object boundaries
  const firstBrace = stripped.indexOf('{');
  const lastBrace = stripped.lastIndexOf('}');
  if (firstBrace !== -1 && lastBrace !== -1) {
    try { return JSON.parse(stripped.substring(firstBrace, lastBrace + 1)); } catch(e) {}
  }
  
  // Find JSON array boundaries
  const firstBracket = stripped.indexOf('[');
  const lastBracket = stripped.lastIndexOf(']');
  if (firstBracket !== -1 && lastBracket !== -1) {
    try { return JSON.parse(stripped.substring(firstBracket, lastBracket + 1)); } catch(e) {}
  }
  
  return null; // truly malformed
}
```
